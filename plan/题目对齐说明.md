## 一、通读要求（每人）

1. 按 **1 → 2 → 3 → 4** 顺序完整读一遍原文（约 1～2 小时）。
2. 读时标注：**本版新增规则**、**与上一版冲突处**、**输出格式容易写错的地方**。
3. 对齐会上每人准备：**用 3～5 句话说清「这一版比上一版多了什么」**。

---

## 二、四版本共性（四题一致的部分）

- 红/蓝司令部、五种武士：`dragon`、`ninja`、`iceman`、`lion`、`wolf`。
- **双方第 n 个武士编号都是 n**（红方第 n 个编号 n，蓝方第 n 个也是 n）。
- **制造顺序（循环）**一致：
  - 红：`iceman → lion → wolf → ninja → dragon`
  - 蓝：`lion → dragon → ninja → iceman → wolf`
- 造兵消耗司令部「生命元」，消耗量等于该武士初始生命值。

---

## 三、版本递进（组长对齐用）

### 版一：备战

- **范围**：仅从 0 时起不断「造兵」，直到双方司令部都停止造兵；无城市地图、无战斗。
- **造兵**：当前顺位武将若生命元不够，**跳过该类型，尝试下一个**；若五种都无法造，则该方停止造兵。
- **输出**：时间单位为「小时」三位数（如 `000`）；事件文案较长（含 `born with strength ...` 等）。
- **同刻顺序**：先红后蓝。

### 版二：装备

- 在版一基础上，降生时增加 **武器与武士特性**（`sword` / `bomb` / `arrow`，dragon 士气、lion 忠诚度等），降生后多行附加输出。
- **造兵逻辑与版一相同**（不够则试下一个）。
- **同刻顺序**：先红后蓝。

### 版三：开战

- 引入 **城市链**（城市 `1…N`，红司令部为 `0`，蓝为 `N+1`）、**生命值 / 攻击力**、**武器与完整战斗流程**。
- **造兵规则与版一 / 二不同**：若轮到某类型但生命元不够，**从此方停止造兵**（不再尝试下一种）。
- **时间轴**（分）：`:00` 降生，`:05` lion 逃跑，`:10` 前进，`:35` wolf 抢武器，`:40` 战斗，`:50` 司令部报告生命元，`:55` 武士报告武器。
- **同刻顺序**：按地点从西向东；同一城市同刻先红后蓝；占领与抵达等细则见原题。
- **注意**：题面武士类型拼写有一处为 `ninjia`，以原题描述为准实现。
- **浮点**：提示中建议避免直接用 `5 * 0.3` 等写法，用整数运算减少精度问题。

### 版四：终极版

- 在版三世界观上规则更复杂：**城市产生命元、拾取与上缴、旗帜与先手、战斗胜负奖励、arrow / bomb 新逻辑、sword 变钝、dragon 士气变化、lion 忠诚度与战死转移生命等**。
- **造兵**：生命元不足时 **不跳过类型**，而是 **等到有足够生命元后的下一个整点** 再制造该顺位兵种。
- **胜负条件**：司令部被占领条件等为 **2 个敌人** 进入（与版三「出现敌人」表述不同，以版四原文为准）。
- **事件类型与输出**种类更多，时间轴含 `:20`、`:30`、`:35`、`:38`、`:40` 等，须严格对照原题。

## 四、易混点（对齐时必须说清）

| topic | 版一 / 版二 | 版三 | 版四 |
|--------|-------------|------|------|
| 生命元不够时是否试下一个兵种 | 是 | 否（直接停造） | 否；等到够的下一个整点再造该顺位 |
| 时间格式 | 三位小时 | `时:分` | `时:分` |
| 地图与城市 | 无 | 有 | 有 |

## 五、对齐完成标准（验收）

- [ ] 三人能口头说出：**版一与版二、版二与版三、版三与版四** 各一处「最关键的规则差异」。
- [ ] 对「造兵失败时怎么办」能明确说出 **哪一版跳过、哪一版停止、哪一版等待整点**。
- [ ] 承认版三、版四细节多，**实现时以 `problems` 原文为准**，AI 教练与样例不编造规则。

## 六、建议流程

1. 每人独立通读 `problems/*.md`（按版本顺序）。
2. 组长组织一次对齐会（约 30～60 分钟），对照本文第三节、第四节核对理解。
3. 对齐后将结论同步到功能设计文档或群公告，便于 Prompt、任务清单、`samples.json` 保持一致。
