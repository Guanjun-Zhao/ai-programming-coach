# AI 陪练教练：提示词设计方案

## 项目背景

本地运行的 C++ 编程陪练工具，调用 Deepseek API，三列布局：左侧任务进度栏、中间代码保存区、右侧 AI 对话框。AI 依据预先准备的参考文档，逐节引导用户完成代码书写。

---

## 核心数据结构

开发时将参考文档按小节预先切割，每节存为一个结构体：

```
{
  section_id:  "1.2.2",
  description: "（该节的自然语言说明，直接来自参考文档）",
  code:        "（该节的参考代码块）"
}
```

两个字段在不同阶段分别使用，互不干扰。

---

## 两个工作阶段

### 阶段一：教学阶段

**触发时机**：用户进入某一节，或在对话框中提问。

**喂给 AI 的内容**：仅该节的 `description`，不包含 `code`。

**AI 的任务**：用自然语言描述这一节需要实现的逻辑，回答用户问题，但不输出任何代码块。

**System Prompt 关键指令**：

> 你是一名 C++ 编程教练。你手中有当前小节的逻辑说明，但没有参考代码。你的教学方式是用中文自然语言描述"这段代码需要做什么事"、"这里的逻辑是如何流动的"、"这个变量存的是什么信息"。不要输出代码块。如果用户要求你直接给出代码，回答"这部分需要你自己动手写，我可以继续描述细节帮你理解"。

---

### 阶段二：验证阶段

**触发时机**：用户在对话框中粘贴自己写好的代码。

**喂给 AI 的内容**：该节的 `code`（参考代码块） + 用户提交的代码。

**AI 的任务**：判断用户代码与参考代码的**逻辑是否等价**（不要求字面一致，变量名、风格不计）。

**System Prompt 关键指令**：

> 以下是本节的参考代码（内部资料，不得输出给用户）和用户提交的代码。请判断用户代码是否实现了与参考代码等价的逻辑功能。只判断逻辑正确性，不评价代码风格。
>
> - 如果通过：输出"你的代码已完成这部分的功能。输入"下一步"，让我们继续下一节的书写。"
> - 如果未通过：用自然语言指出缺少或有误的逻辑点，不要给出正确代码。

---

## 完成流程（用户侧）

验证通过后，用户手动完成以下操作：

1. 将本节代码追加/更新到中列的**统一代码保存区**（中列始终保存完整程序，不按节拆分）；
2. 勾选左侧任务栏对应节的复选框，标记为完成。

AI 不参与这两步，避免自动化带来的复杂性。

---

## 阶段三：Debug 阶段

**触发时机**：用户在所有编码节完成后，点击左侧任务栏末尾的「Debug」按钮。

**前提条件**：中列代码保存区已保存完整的程序代码。

**交互流程**：

1. AI 从后台预置的 `samples.json` 中取出第一条样例，将样例**输入**发送给用户，要求用户在本地运行并将输出结果回传。
2. 用户本地运行后，将实际输出粘贴回对话框。
3. AI 将用户输出与 `samples.json` 中该条样例的正确输出进行**字符串比对**：
   - **一致**：回复"此样例通过"，自动进入下一条样例，重复流程。
   - **不一致**：进入错误分析流程（见下）。
4. 所有样例全部通过后，AI 回复完成提示，用户勾选左侧「Debug」复选框。

**错误分析流程**（输出不一致时）：

AI 此时临时读取中列代码保存区的**完整程序代码**，结合当前样例的输入、期望输出、用户实际输出，分析逻辑错误位置，用自然语言引导用户修改。修改后用户重新运行，将新输出回传，AI 重新比对。

**System Prompt 关键指令（Debug 阶段）**：

> 你是一名 C++ 调试教练。你将逐条给用户提供测试样例输入，用户在本地运行后将结果回传给你。你手中有每条样例的正确输出，请逐字符比对用户的实际输出与正确输出。
> - 如果一致：回复"样例 N 通过，下面是第 N+1 条样例："并给出下一条样例输入。
> - 如果不一致：结合用户完整程序代码（见下方）、样例输入、期望输出和实际输出，用自然语言分析哪段逻辑可能导致了这个差异，引导用户排查和修改。不要直接给出修改后的代码。

---

## 关键设计决策

| 问题 | 决策 |
|------|------|
| 教学时是否给 AI 看参考代码？ | 否，只给 description |
| 验证时用 code 还是 description 做基准？ | 用 code 块，判断更准确 |
| 泄露风险如何控制？ | System Prompt 明确标注为内部资料，禁止输出 |
| 任务勾选和代码更新是否自动化？ | 否，全部由用户手动完成 |
| AI 完成信号如何设计？ | 输出固定自然语言提示，无需机器可读标记 |
| 中列代码保存方式？ | 统一保存完整程序，不按节拆分 |
| Debug 阶段样例比对方式？ | 字符串精确比对；不一致时 AI 读取完整代码做逻辑分析 |
| Debug 阶段 AI 是否直接给修改代码？ | 否，只用自然语言引导排查 |
