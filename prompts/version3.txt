## 结构化规则（版三：开战）

### 地图与司令部

- 城市自西向东编号 `1 … N`（N ≤ 20）；红司令部为城市 **0**，蓝司令部为城市 **N+1**。司令部有生命元用于造兵。
- 武士生命值为非负；归零（负数按 0 处理）则死亡消失。
- 武士到达**敌方司令部**后停留；敌方司令部内若出现敌人则视为可被占领（详见题面）。

### 武士与编号

- 五种：`dragon`、`ninja`、`iceman`、`lion`、`wolf`；各有编号、生命值、攻击力。
- 红方第 n 个与蓝方第 n 个武士编号均为 n。

### 武器（编号 0/1/2 → sword/bomb/arrow）

- **sword**：攻击力 = 使用者当前攻击力的 20%（去尾取整）。
- **bomb**：对敌攻击力 = 使用者当前攻击力的 40%（去尾取整）；使用者也会受到伤害（题面公式）；用后即无。
- **arrow**：攻击力 = 使用者当前攻击力的 30%（去尾取整）；一支 arrow 可用 **两次**。

### 战斗规则（核心）

1. **先手**：**奇数**编号城市红方先攻；**偶数**编号城市蓝方先攻。
2. **战前**：双方各自将武器按使用顺序排好，再按顺序一件一件用；**编号小的武器类型排在前面**；多支 arrow **用过的排在前面**。
3. **使用轮流**：双方轮流使用一件武器；一方一轮用尽则从头再轮转；若一方无武器则可能挨打直至结局。
4. **平局**：双方武器都用尽仍都存活 → 平局；双方都死 → 平局；武器皆为 0 仍要打；若状态不再变化 → 平局。
5. **胜负与缴获**：胜方获得败方武器；每人手持武器 **不超过 10**；缴获时按武器种类编号从小到大；多 arrow 优先缴**未用过**的。
6. **无前武器**：若开战前两方都无武器 → 平局；若先手方无武器而后手有 → 由后手攻击（题面规则）。
7. **dragon**：编号 n 降生得武器 **n%3**；战后若未死则**欢呼**。
8. **ninja**：武器 **n%3** 与 **(n+1)%3**；使用 bomb **不自伤**（题面表述；原文有一处类型名为 `ninjia`，实现与输出类型名以 **题目原文与评测约定** 为准）。
9. **iceman**：武器 **n%3**；每**前进一格**生命值减少当前值的 **10%**（去尾取整）。
10. **lion**：武器 **n%3**；忠诚度初值 = 降生后司令部剩余生命元；每前进一格忠诚度减 **K**；≤0 则逃跑（已到达敌方司令部的 lion 不逃）；可能在己方司令部逃跑。
11. **wolf**：降生无武器；在 **:35** 可与敌人抢夺武器（见事件）；若双方均为 wolf **不抢**；抢夺规则见题面（编号最小种类、多件 arrow 优先未用等）。

### 制造顺序与造兵（版三）

- 红：`iceman` → `lion` → `wolf` → `ninja` → `dragon`；蓝：`lion` → `dragon` → `ninja` → `iceman` → `wolf`。
- 生命元不足则 **停止造兵**（不顺延下一兵种）。

### 其他全局规则

- 占领：任一方司令部出现敌人即被占领；战争结束则不再发生任何事件。

### 时间轴（同一「模拟小时」内）

- **:00** 双方司令部各降生一名武士（若仍可造）。
- **:05** 应逃跑的 lion 逃跑。
- **:10** 所有武士向敌司令部方向**前进一步**。
- **:35** 有 wolf 与敌人的城市：wolf 抢夺武器。
- **:40** 有两名武士的城市：**战斗**。
- **:50** 双方司令部报告各自生命元数量。
- **:55** 每个武士报告武器情况。

### 输出时间格式

- `HHH:MM`，时三位、分两位（如 `000:40`）。

### 事件类型与字符串要点（浓缩）

- **降生**：`000:00 blue dragon 1 born`；lion 额外一行 `Its loyalty is 忠诚度`。
- **lion 逃跑**：`… lion … ran away`。
- **前进**：`red/blue 类型 编号 marched to city … with 生命值 elements and force 攻击力`（iceman 输出**变化后**生命值）。
- **wolf 抢武器**：`wolf … took … from … in city …`。
- **战斗结果**：杀死 / 双死 / 双活等格式见题面；红武士写在前面。
- **dragon 欢呼**：`… dragon … yelled in city …`。
- **抵达敌方司令部**：`reached … headquarter with …`。
- **占领**：`… headquarter was taken`。
- **司令部生命元**：`… elements in red/blue headquarter`（:50）。
- **武器报告**：`… has … sword … bomb … arrow … and … elements`（次序 **sword、bomb、arrow**）。

### 输出顺序

- 按时间；同刻按地点 **自西向东**；前进算发生在**目的地**。
- 同城市同刻：先红后蓝（题面细则）。
- 同一场战斗内多个子事件：编号小者优先；一次战斗中可有杀死 / 欢呼等多种事件同时刻，按题面给定序号顺序输出。
- **抵达司令部**与**司令部被占领**若同时刻，先输出抵达，再输出占领。

### 输入格式

- 第一行 t。
- 每组三行：  
  - `M N K T`：初始生命元 M、城市数 N、lion 每步忠诚衰减 K、截止时刻 T（**分钟**，含 T）。  
  - 五种**初始生命值**：dragon、ninja、iceman、lion、wolf。  
  - 五种**攻击力**：同上顺序。

### 输出格式

- 每组首行 **`Case n:`**（注意空格）。
- 随后输出到时刻 T 为止（含 T）的所有事件。

### 实现提示（题面原文）

- 避免写 `5 * 0.3` 这类浮点；改用整数运算如 **`5 * 3 / 10`** 等以减少评测机差异。
